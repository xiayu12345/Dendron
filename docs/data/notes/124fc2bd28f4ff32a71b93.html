BK7252n LVGL：桌面、二维码页与 MQTT 图片预览 UI 开发总结

日期：2026-05-07
工程：bk7252n_sdk-SDK_3.0.76.1_witch_lcd
说明：本文汇总本轮在二级桌面、状态栏、模式卡片、语音卡片、静态二维码页及 WebSocket 热敏预览等方向的实现与参数约定，便于后续维护与评审。

1. 编译与环境

  • 推荐命令：scons --beken=bk7252n -j4
    plain scons 可能选错芯片库，出现缺头文件等问题。
  • 产物：打包后在 out/ 目录（如 rtthread_uart_4M.*.bin、all_4M.*.bin）。

2. OTA（与本轮 UI 并行存在的能力）

  • 配置：config/fal_cfg.h（OTA 分区表、编译包含 -Iconfig）。
  • 应用：applications/OTA_Upgrade/，rtconfig.h 中 PKG_USING_WEBCLIENT、PKG_USING_RT_OTA 等。
  • MQTT：applications/mqtt_backend 侧包含 OTA_Upgrade.h，支持上层触发升级流程。

（详细接线以同事已验证方案为准，本文不展开协议细节。）

3. 桌面状态栏

  • 透明：状态栏背景与桌面壁纸协调（半透明/透明相关样式在 lv_example_meter.c 中维护）。
  • 时间占位：为避免开机短暂 --:-- 带来的「拖沓感」，初始与 RTC 未就绪时统一显示 12:00。
    • 宏：UI_STATUS_TIME_PLACEHOLDER
    • 逻辑：status_bar_build 初始文案 + status_time_refresh() 中 time 无效分支。

4. 六个模式图标与底部文案

  • 模式浅色底卡片：150×150，LV_RADIUS_CIRCLE 裁成圆（与语音副卡视觉语言对齐）。
  • 模式标题（动漫 / 卡通等）：使用 mode_common_label()，LV_ALIGN_BOTTOM_MID 垂直偏移由宏 MODE_PANEL_LABEL_BOTTOM_Y_OFS 控制（当前为 -17，相对原先 -2 整体上移 15px，减轻贴底观感）。

5. 录音 / 语音二级界面

  • 卡片尺寸：150×150，圆形（LV_RADIUS_CIRCLE），与桌面当前模式壳一致。
  • 布局：录音与「生成中」共用 voice_panel_icon_slot_layout + voice_panel_footer_labels。
    • 使用 VOICE_PANEL_ICON_EXTRA_UP_RECORD_GEN 上移图标区，避免圆形底部裁切文案。
    • Footer 内文案对齐改为 LV_FLEX_ALIGN_START（与错误页一致），避免 END 把字挤到圆弧底部。
  • 圆角矩形已废弃：由原先 135×135、圆角 14 改为全圆 150×150。

6. 静态二维码页（P36 长按等）

  • 面板外框：210 × THERMAL_QR_FRAME_H（默认 210×204，由 thermal_happy_lv.h 中 SN 行高 + 二维码逻辑框 + 底部说明高度相加）。
  • 面板背景：浅黄 #FFF9C4（UI_DESKTOP_CARD_BG_HEX），半透明 UI_QR_PANEL_BG_OPA（LV_OPA_70），圆角 UI_QR_PANEL_RADIUS（8）。
  • SN 行：不再使用单独灰色条；SN 与底部提示 使用深色字（如 #101010 / #303030）保证在浅黄底上可读。
  • 垂直位置：宏 THERMAL_QR_PANEL_CENTER_Y_OFS（默认由 20 调整为 8，累计上移），整块相对屏幕居中基准下移量减少。

7. MQTT WebSocket 热敏图片预览（二级桌面 UI_RESULT_WS_IMAGE）

7.1 结构

  • s_sec_ws_shell：lv_obj_create(s_ui_layer)，圆形浅黄半透明底（与二维码面板同色同透明度约定），clip_corner 裁剪子控件。
  • s_sec_img：挂在 s_sec_ws_shell 内，LV_ALIGN_CENTER，数据源仍为 thermal_happy_get_img_dsc()。

7.2 位置

  • 与时间栏对齐：紧贴状态栏下沿。
    • 状态栏：LV_ALIGN_TOP_MID, 0, 1，高度 UI_DESKTOP_STATUS_BAR_H（25）。
    • 预览壳：LV_ALIGN_TOP_MID, 0, UI_WS_PREVIEW_SHELL_TOP_Y，其中
      UI_WS_PREVIEW_SHELL_TOP_Y = 1 + UI_DESKTOP_STATUS_BAR_H（即 26）。

7.3 尺寸（当前统一约定）

  • THERMAL_WS_PREVIEW_BOX_W / THERMAL_WS_PREVIEW_BOX_H（applications/images/thermal_happy_lv.h）：245×245
    • 作为 thermal_happy_set_ws_image_fit(..., max_w, max_h) 的上限，图源 384 宽 1bpp 等比缩放装入框内。
  • UI_WS_PREVIEW_SHELL_W / UI_WS_PREVIEW_SHELL_H（lv_example_meter.c）：等于 THERMAL_WS_PREVIEW_BOX_*，保证容器与解码上限一致，避免一圈空白或裁切歧义。

历史迭代简述（便于对照 git）：202×202 壳 → 220 壳 + 235 解码 → 统一 284 → 统一 245（当前）。

7.4 内存与主流程

  • 圆形底为 LVGL 样式绘制，不增加 245×245 全屏位图缓存；解码缓冲仍在 thermal_happy_lv.c 既有 PREVIEW_LV_BUF 路径内（按 384 规模预留），245×245 远低于上限。
  • MQTT / 打印主流程不因 UI 壳改变；需注意的仍是历史上讨论的 g_binary_shared_buf 与异步预览指针生命周期（见 BUG-001：按键极限操作触发 Data Abort / 意外重启）。

8. 工具脚本（资产与桌面图）

  • tools/scripts/gen_desktop_bmp.py：可选 --strip 等，用于替换桌面 BMP 顶部条、再生 desktop_bmp.c / BMP_IMG/Desktop.bmp。
  • tools/scripts/gen_mode_icons_bmp.py：与桌面色 #FFF9C4（MATTE_RGB）一致的模式图标资产（README 或脚本内说明）。

10. 后续可选优化（非本轮必做）

  • 预览与打印并发时，对 WS 预览快照 memcpy 或串行化访问 g_binary_shared_buf，降低偶发花屏/解析失败概率。
  • 网络「假连接」判定：在 bk_wlan_sta_is_connected / IP 之外增加 Ping 网关或 MQTT 可达性（见工程内 WiFi 相关讨论）。

文档随固件迭代时请同步更新「第 7.3 节尺寸」与相关宏默认值。
