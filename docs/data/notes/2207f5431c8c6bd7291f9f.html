1. 背景

语音生成中页面（VOICE_UI_GENERATING 状态）此前使用 LVGL 原生绘制的卡通小熊 + 上下弹跳动画作为等待指示。效果粗糙、生硬，且 LVGL 原生几何体绘制能力有限，无法呈现丰富的角色细节。

为提升产品"陪伴感"，需要一个更可爱的动画角色在 AI 生成语音内容期间展示，目标受众为 6-10 岁儿童。经过方案对比（小狗/小猫/恐龙/福星），最终选定 小机器人，因其方形几何结构在像素画中表现力最佳，且契合 AI 打印机产品调性。

3. 技术方案演进

3.1 方案一：LVGL 原生绘制（小熊 → 恐龙）

最初尝试用 lv_obj_create + lv_palette_main 等 LVGL 原生 API 绘制卡通角色。先后尝试了小熊和恐龙，效果均不佳——几何体拼接感强，圆形/圆角矩形组合无法呈现"可爱"质感。

结论：放弃原生绘制路线，改用像素图片。

3.2 方案二：BMP RGB565 逐帧

用 Python 脚本生成 4 帧 80×80 的 BMP RGB565 图片，编译进固件后用 lv_img_set_src 轮换。

问题：

  • Flash 占用大：80×80×2 字节/像素 ≈ 12.8 KB/帧，4 帧 ≈ 51 KB
  • 透明处理依赖 chroma-key：需将背景设为 LV_COLOR_CHROMA_KEY 纯绿（0x00FF00），LVGL 渲染时跳过该颜色。实际出现过品红背景（因脚本颜色常量写错）、白色卡片上透明色差等问题

结论：可用但 Flash 不友好，chroma-key 调试麻烦。

3.3 方案三（最终方案）：LV_IMG_CF_INDEXED_4BIT

LVGL v8 内置 4-bit 索引色格式：

  • 每像素仅 4 bit（2 像素打包为 1 字节）
  • 前 64 字节为 16 色调色板（RGBA8888 格式），之后为打包像素 nibble
  • 天然支持 alpha 透明：调色板中背景色 alpha=0x00，其余颜色 alpha=0xFF
  • 无需 BMP 解码器，LVGL 直接渲染

数据布局：

[调色板 16×4=64B] [像素数据 (56×68+1)/2=1904B] = 1968B/帧

4 帧总计 ~7.9 KB，比 BMP 方案省 ~84% Flash。

尺寸优化：80×80 → 56×68，裁剪掉上下无用透明区域，进一步节省。

背景色：白色（0xFFFFFF），与语音面板 150×150 白色圆形卡片同色融合。

4. 详细改动

4.1 gen_dino_frames.py — 像素画生成脚本

4.1.1 画布与调色板

W, H = 56, 68  # 最终尺寸

C_TRANS  = rgb565(0xFF, 0xFF, 0xFF)  # 背景白 → alpha=0
C_BODY   = rgb565(0x90, 0xCA, 0xF9)  # 浅蓝身体
C_BODY_D = rgb565(0x5C, 0x9C, 0xE6)  # 深蓝（腿/天线杆）
C_FACE   = rgb565(0xE0, 0xE0, 0xE0)  # 灰白脸盘
C_ANTENNA= rgb565(0xF4, 0x43, 0x36)  # 红色天线球
C_EYE_BG = rgb565(0xFE, 0xFF, 0xFF)  # 近白眼白（与背景白区分，确保 alpha=0xFF）

背景纯白 0xFFFFFF 与卡片底色一致；眼白用 0xFEFFFF 微妙区分，确保索引色调色板中眼白 alpha 不被误设为 0。

4.2 dino_frames_bmp.h — 帧表头文件

#define DINO_FRAME_COUNT 4

extern const lv_img_dsc_t DINO_IDLE_BMP;
extern const lv_img_dsc_t DINO_BLINK_BMP;
extern const lv_img_dsc_t DINO_WIGGLE_BMP;
extern const lv_img_dsc_t DINO_HAPPY_BMP;

static const lv_img_dsc_t *const s_dino_frame_table[DINO_FRAME_COUNT] = {
    &DINO_IDLE_BMP,
    &DINO_BLINK_BMP,
    &DINO_WIGGLE_BMP,
    &DINO_HAPPY_BMP,
};

帧表 s_dino_frame_table 为 static const，在包含此头文件的每个 .c 中独立实例化，避免全局符号冲突，同时允许编译器内联优化。

4.3.2 新增静态变量

/* Dino frame animation */

static uint8_t s_dino_frame_idx;

4.3.3 帧切换回调

static void dino_frame_timer_cb(lv_timer_t *t)
{
    lv_obj_t *img = (lv_obj_t *)t->user_data;
    s_dino_frame_idx = (uint8_t)((s_dino_frame_idx + 1) % DINO_FRAME_COUNT);
    lv_img_set_src(img, s_dino_frame_table[s_dino_frame_idx]);
}

周期 200ms，4 帧循环一次约 800ms。

4.3.4 替换绘制函数

voice_panel_generating_cartoon_pet() 由原来的 LVGL 原生小熊绘制替换为：

static void voice_panel_generating_cartoon_pet(lv_obj_t *slot)
{
    lv_obj_t *pet;
    lv_obj_t *img;

    pet = lv_obj_create(slot);
    lv_obj_remove_style_all(pet);
    lv_obj_set_size(pet, 56, 68);
    lv_obj_set_style_bg_opa(pet, LV_OPA_TRANSP, 0);
    lv_obj_clear_flag(pet, LV_OBJ_FLAG_SCROLLABLE);
    lv_obj_add_flag(pet, LV_OBJ_FLAG_OVERFLOW_VISIBLE);
    lv_obj_center(pet);

    img = lv_img_create(pet);
    lv_img_set_src(img, s_dino_frame_table[0]);
    lv_obj_center(img);

    s_dino_frame_img = img;
    s_dino_frame_idx = 0;

    /* Frame cycle: ~200ms per frame */
    s_dino_frame_timer = lv_timer_create(dino_frame_timer_cb, 200, img);

    /* Keep the gentle bounce on the pet root */
    s_vpanel_voice_anim_obj = pet;
    voice_panel_start_generating_bounce(pet);
}
  • 容器尺寸：56×68（与图片尺寸一致）
  • 弹跳动画：继承原有 voice_panel_start_generating_bounce，translate_y 在 0 ? -6 之间缓动，周期 ~1s
  • 两层动画叠加：容器弹跳 + 内部帧切换，形成"蹦蹦跳跳的机器人"效果

4.3.5 清理逻辑增强

static void voice_anim_cleanup(void)
{
    if (s_vpanel_voice_anim_obj != NULL) {
        lv_anim_del(s_vpanel_voice_anim_obj, NULL);
        s_vpanel_voice_anim_obj = NULL;
    }
    if (s_dino_frame_timer != NULL) {
        lv_timer_del(s_dino_frame_timer);
        s_dino_frame_timer = NULL;
    }
    s_dino_frame_img = NULL;
    s_dino_frame_idx = 0;
}

在原有 bounce 动画清理基础上增加帧 timer 销毁和图片指针置空。voice_anim_cleanup 在以下场景被调用：

  • 语音面板重建（voice_panel_rebuild）
  • 语音状态切换（voice_ui_apply_under_lock）
  • 界面销毁（ui_destroy_layers）
  • 二级桌面刷新（second_desktop_refresh_center）

确保离开生成状态后 timer 立即停止，无内存泄漏和野指针。

8. 注意事项

  • 修改帧图：编辑 gen_dino_frames.py 中的 draw_robot 函数，运行 python3 gen_dino_frames.py 重新生成 .c/.h，再编译即可。无需手动编辑 C 数组。
  • 调整帧速：修改 lv_timer_create(dino_frame_timer_cb, 200, img) 中的 200（毫秒）。值越小越快。
  • 颜色限制：INDEXED_4BIT 最多 16 色（含透明）。当前 12 色，有 4 色余量可添加新颜色。
  • 设计师替换：如需设计师提供像素画，可提供 56×68 的 PNG 模板，由脚本导入替代 draw_robot 生成逻辑；或设计师直接出 INDEXED_4BIT 格式的调色板+像素数据替换 dino_frames_bmp.c。
