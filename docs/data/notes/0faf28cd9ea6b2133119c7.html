LVGL 模式卡片、六模式图标资源与动效 — 开发总结

本文汇总本会话周期内针对 BK7252n + LVGL 主桌面六模式 的相关开发与排查结论。
实现细节以当前仓库内源码为准（例如 MAIN_MODE_PANEL_W/H、动效开关默认值可能随后续提交调整）。

1. 六模式图标资源（PNG → BMP → 固件）

1.1 文件位置

用途路径
源 PNG（可按模式替换）applications/images/mode_jpg/
生成脚本tools/scripts/gen_mode_icons_bmp.py
嵌入固件的 C 数组applications/images/mode_icons_bmp.c
对外声明applications/images/mode_icons_bmp.h

1.2 生成规则（脚本）

  • 顺序：与 lv_example_meter.c 中 mode_icon_dsc_for_idx() 一致 — 动漫 → 卡通 → 水墨画 → 线条画 → 简笔画 → 素描。
  • 命名回退：每个模式优先 <stem>.png，若无则用 <stem>1.png（便于目录里混用「动漫.png / 动漫1.png」）。
  • 尺寸：统一缩放为 100×100，输出 24 位 BMP，在 lv_img_dsc_t 中使用 LV_IMG_CF_RAW（整文件嵌入，由 LVGL BMP 解码）。
  • 透明通道：工程 LV_COLOR_DEPTH 为 16 时，嵌入式 BMP 路径下难以做到真正意义上的 Alpha 合成；脚本对 RGBA 图源采用 #FFF9C4（与桌面黄卡片同色） 做垫色合成，避免白边；纯 RGB 图源不做垫色。

脚本内常量 MATTE_RGB = (255, 249, 196) 应与桌面卡片背景一致（如 UI_DESKTOP_CARD_BG_HEX / 0xFFF9C4）。

1.3 重新生成命令

python3 tools/scripts/gen_mode_icons_bmp.py
scons --beken=bk7252n -j8

2. 模式卡片 UI（黄底 + 圆形）

2.1 样式要点

  • 背景色：浅黄 #FFF9C4。
  • 形状：正圆 — 宽与高相等时使用 LV_RADIUS_CIRCLE，并配合 lv_obj_set_style_clip_corner(..., true)，避免子控件画出方角。
  • 两处需一致：
    • main_mode_shell_style：主桌面当前模式外壳。
    • voice_panel_attach_card：语音流程里同风格衬底卡片。

2.2 尺寸说明（以源码宏为准）

会话中曾用 135×135；当前仓库若为 150×150 等，以 MAIN_MODE_PANEL_W / MAIN_MODE_PANEL_H（及 VOICE_CARD_PANEL_*）为准。

图标 hub 典型为 MAIN_MODE_PANEL_W - 8 × 102，图标 100×100，标题底部对齐（如 MODE_PANEL_LABEL_BOTTOM_Y_OFS）。

3. 模式图标动效（LV_MODE_PANEL_ICON_ANIM）

3.1 开关

  • 宏：LV_MODE_PANEL_ICON_ANIM（lv_example_meter.c）。
  • 1：对当前 s_mode_fx_hub[idx] 启动 lv_anim（具体组合见 mode_panel_img_anim_start()）。
  • 0：编译期裁掉动效实现。
  • 可 -DLV_MODE_PANEL_ICON_ANIM=0 覆盖。

3.2 本会话演进（对照）

  1. 曾关动效 / 再开，并加大位移、缩短周期便于观察。
  2. 曾改为全员 TY 弹跳 + TX 摆动、去掉透明度呼吸（若仓库后续又混用透明度，以当前 switch (idx) 为准）。

3.3 行为说明

  • 同一时间只显示 当前模式 一张卡片，动效只作用当前 idx 的 hub。
  • 动效作用在 hub，减轻对 lv_img transform 与 layer RAM 的压力。

4. 开机花屏（雪花 / GRAM 未初始化）

  • 原因：GRAM 随机内容被扫出或背光过早。
  • beken378/driver/lcd/lcd_driver.c：drv_lcd_open() 中注释建议 _lcd_clear(COLOR_BLACK) 后再亮背光（是否启用看该行是否仍注释）。
  • applications/main.c：关背光 → drv_lcd_open() → LVGL/UI → 再开背光；仍花屏时核对背光 GPIO 极性与其它提前拉高背光的代码。

5. 附录（本会话问答）

主题要点
Flash 余量分区大小见 config.json / tools/scripts/post_action.py；wc -c rtthread.bin 得镜像大小；余量 ≈ 分区 ? bin（视对齐而定）。
heap / memheap / ITCMheap 为动态内存概念；memheap 为 RT-Thread 堆实现；ITCM 见 link.lds，与 malloc 堆不同。
CPU 占用list_thread / ps 看线程与栈占用百分比；整 CPU 可参考 rt-thread/examples/kernel/cpuusage.c 自行挂 Shell。
