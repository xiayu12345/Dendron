2026-05-15 首页六模式隐藏功能开发记录

需求背景

首页六个打印模式（动漫、卡通、水墨画、线条画、简笔画、速写）中，动漫、水墨画、简笔画三个模式打印效果不佳，需要暂时从首页轮播中隐藏。但代码不能删除，后续打印效果改善后需要恢复显示。

涉及文件

  • applications/pages/lv_example_meter.c

变更内容

新增隐藏模式掩码宏

在 UI_STATUS_TIME_PLACEHOLDER 定义之后、ui_mode_next_visible 函数之前，新增：

/*
 * Bitmask of mode indices hidden from desktop rotation.
 * Bit 0=anime, 1=cartoon, 2=ink_wash, 3=line_art, 4=simple_drawing, 5=sketch.
 * Set to 0 to show all six modes; set (1<<0)|(1<<2)|(1<<4) to hide anime/ink/simple.
 */
#ifndef LV_DESKTOP_HIDDEN_MODE_MASK
#define LV_DESKTOP_HIDDEN_MODE_MASK ((1U << 0) | (1U << 2) | (1U << 4))
#endif

隐藏的模式： | 位 | 索引 | 模式名 | 枚举键 | |----|------|--------|--------| | 0 | 0 | 动漫 | anime | | 2 | 2 | 水墨画 | ink_wash | | 4 | 4 | 简笔画 | simple_drawing |

保留的模式：卡通(1)、线条画/线稿(3)、速写/素描(5)

新增辅助函数

static uint8_t ui_mode_next_visible(uint8_t cur)
{
    uint8_t i;
    for (i = 0; i < MODE_SLOT_COUNT; i++) {
        cur = (uint8_t)((cur + 1) % MODE_SLOT_COUNT);
        if (!(LV_DESKTOP_HIDDEN_MODE_MASK & (1U << cur)))
            return cur;
    }
    return cur;
}

从当前索引出发，循环查找下一个未被掩码标记的可见模式。

修改模式切换逻辑

desktop_next_cb() 中：

// 旧代码
s_mode_idx = (uint8_t)((s_mode_idx + 1) % ui_mode_count());

// 新代码
s_mode_idx = ui_mode_next_visible(s_mode_idx);

修改初始化逻辑

lv_example_meter() 中，如果初始索引 0 在隐藏列表中，则自动跳到第一个可见模式：

s_mode_idx = 0;
if (LV_DESKTOP_HIDDEN_MODE_MASK & (1U << s_mode_idx))
    s_mode_idx = ui_mode_next_visible(s_mode_idx);

恢复方法

方法一：改代码（推荐）

将 lv_example_meter.c 中的掩码改为 0：

#define LV_DESKTOP_HIDDEN_MODE_MASK 0

方法二：编译时覆盖

scons --beken=bk7252n -j4 CCFLAGS="-DLV_DESKTOP_HIDDEN_MODE_MASK=0"

影响范围

  • 仅影响首页模式轮播 UI：用户按键切换模式时跳过隐藏模式
  • 不影响打印机驱动：thermal_printer.c 通过模式名字符串匹配参数，不受掩码影响
  • 不影响多语言字符串：ui_strings.c 中的模式名称翻译全部保留
  • 不影响图标资源：mode_icons_bmp.c 中的六个图标全部保留

编译验证

scons --beken=bk7252n -j4
# 编译通过，无错误无警告

备注

此方案采用掩码位图方式控制隐藏，扩展性好：

  • 如需隐藏其他模式，只需设置对应位
  • 支持非连续性隐藏（当前隐藏 0、2、4）
  • 不影响代码中其他任何对模式索引/名称的引用
