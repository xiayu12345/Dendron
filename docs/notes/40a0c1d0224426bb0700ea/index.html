BK7252N 烧录教程（基于当前工程产物）

本文档对应当前工程 bk7252n_sdk-SDK_3.0.76.1_witch_lcd，并基于你刚编译得到的 out 目录产物。

1. 先确认编译产物

成功编译后，out 目录里通常有这些文件：

  • bootloader_bk7252n_uart1_v1.0.14.bin：Bootloader
  • rtthread.bin：应用裸 bin
  • rtthread.elf：调试用 ELF
  • rtthread.map：链接映射
  • all_4M.1220.bin：整包镜像（bootloader + app）
  • rtthread_uart_4M.1220.bin：串口下载镜像（app 分区导出镜像）

推荐优先使用 all_4M.1220.bin 做“全量刷机”，最稳。

2. 烧录工具说明

本 SDK 自带的官方烧录器位于：

  • tools/beken_program_V2.5.4/BekenProgramV2.5.4.exe

这是 Windows GUI 工具。
如果你现在在 Linux 环境开发，通常有三种方式：

  1. 到一台 Windows 电脑上烧录（最常见）
  2. Linux 上用虚拟机跑 Windows 再烧录
  3. Linux + Wine 运行该 exe（兼容性不保证）

3. 硬件连接

按常见 BK7252N 串口下载流程准备：

  1. USB 转串口连接开发板（TX/RX/GND，电平 3.3V）
  2. 确认串口号（Windows 设备管理器中的 COMx）
  3. 板子进入下载模式（通常是按住 Boot/Download 键上电或复位）

不同板卡进入下载模式的按键组合可能不同，按你主板资料为准。

4. 方法 A：全量烧录（推荐，首次或异常恢复）

4.1 打开工具

运行：

  • tools/beken_program_V2.5.4/BekenProgramV2.5.4.exe

4.2 关键配置

在工具中设置：

  • 串口：选择你的 COMx
  • 波特率：先用默认（常见 115200；若工具推荐更高可按工具默认）
  • 下载文件：选择 out/all_4M.1220.bin
  • 下载地址：通常工具对整包模式会自动处理；若需要手填，填 0x00000000

4.3 开始烧录

  1. 点击下载/开始（Start/Download）
  2. 若工具提示等待 Boot，请重新执行“下载模式上电/复位”
  3. 看到成功日志后重启板子

5. 方法 B：分区烧录（仅更新应用）

适用于 Bootloader 不变、只升级应用的情况。

可选文件：

  • out/rtthread_uart_4M.1220.bin（常用于串口升级流程）
  • 或 out/rtthread.bin（原始 app bin）

推荐地址（来自本次打包配置）：

  • bootloader 起始地址：0x00000000
  • app 起始地址：0x00011000

如果工具支持按地址写入：

  • 仅升级应用时，将 app 镜像写到 0x00011000

分区烧录必须确保地址与当前分区表一致，否则会导致无法启动。

6. 烧录后如何验证

  1. 串口打开日志终端（115200 8N1，按工程默认）
  2. 板子复位上电
  3. 观察是否出现 RT-Thread 启动日志和你的应用日志
  4. 执行功能自检（联网、按键、显示、音频等）

7. 常见问题排查

7.1 工具连不上板子

  • 检查 USB 转串口驱动是否正常
  • TX/RX 是否交叉连接（PC-TX 到板-RX）
  • GND 是否共地
  • 是否真的进入下载模式

7.2 烧录成功但不启动

  • 首次请改用“全量烧录”all_4M.1220.bin
  • 检查是否误写了错误地址
  • 检查镜像是否对应芯片型号 bk7252n

7.3 启动后异常重启

  • 查看串口启动日志定位 fault
  • 对照 out/rtthread.elf + out/rtthread.map 做地址回溯

8. 你当前这版的推荐操作

对你当前工程，建议直接执行：

  1. 进入下载模式
  2. 用 BekenProgramV2.5.4.exe 选择 out/all_4M.1220.bin
  3. 地址 0x00000000（若工具要求）
  4. 烧录完成后复位，观察串口日志
