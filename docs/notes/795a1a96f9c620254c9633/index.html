1. 架构评估文档

  • 在 wjz-docs（及后续整理的 0wjz-docs）中新增《AI 打印机应用架构评估与优化方向》类文档，对应用主链路做只读分析。
  • 重点结论方向包括：MQTT/二进制接收/打印/UI/音频/OTA/配网耦合、安全与启动可用性、后续解耦与可观测性等（具体以仓库内同名 md 为准）。

2. 打印速度优化（多轮迭代）

2.1 思路概览

  • 减少无效耗时：去掉重复长走纸、减少固定 rt_thread_mdelay 累计、压缩尾部 feed。
  • 白行加速：空白行快速进纸；连续白行批量合并快进（上限逐步提高，末版为 96 行一批量级）。
  • speed=5 全局参数激进：缩短 line_cooling_us、inter_group_us，下调各温度档 heat_us_at_*，加快 motor_step_us，拉大 status_check_period_lines 降低温度采样频率。
  • 高密度行补偿：在“速度优先”前提下略调低行内黑点加热补偿，平衡黑度与耗时。

2.2 涉及文件（主要）

  • applications/Printer/thermal_printer.c：速度档、位图打印循环、白行路径、perf 页脚等。
  • applications/mqtt_backend/mqtt_backend_binary_rx.c：打印线程内去掉多余等待与重复 feed，避免与 thermal_printer_print_bitmap 内尾部走纸叠加。

2.3 末版（本记录撰写时）speed=5 典型全局参数（供对照）

实际以源码 apply_speed_profile 中 default 分支为准，以下为记录时的量级说明。

参数末版量级说明
inter_group_us1（组间间隔极短）
line_cooling_us10
heat_us_at_5c / 25c / 40c约 1080 / 940 / 820
motor_step_us约 650
status_check_period_lines约 96
连续白行批量上限约 96 行
周期性线程让步约每 256 行 mdelay(1)
位图结束后 feed较短（如 2 行量级）

3. 纸上测速与六种模式状态（便于拍照分析）

3.1 时间戳

  • 在 thermal_printer_print_bitmap 结束后，通过 thermal_printer_print_text 在纸上追加：
    • S:<start_ms> 开始毫秒（基于 rt_tick 换算）
    • E:<end_ms> 结束毫秒
    • C:<cost_ms>ms 耗时

3.2 六种 UI 模式列表

  • 新增 lv_desktop_format_print_footer()（applications/pages/lv_example_meter.c / 声明于 lv_example_meter.h）。
  • 纸上在 Modes: 下列出与 UI 一致的六种模式名，当前选中行前缀 * ，其余 - ：
    • anime、cartoon、ink_wash、line_art、simple_drawing、sketch
  • thermal_printer.c 内使用防递归标志，避免 print_text 再次进入 print_bitmap 时重复打页脚。

4. 按键：长按模式键打印二维码

  • 在 applications/button_control/button_control.c 中，长按 P36 调用 QR_code_printer()。
  • 二维码内容优先使用 device_config_get_sn() 的 SN；失败时回退固定字符串（便于区分设备）。

注意：P37 仍为长按录音进入二级桌面逻辑，与 P36 长按二维码分离。

5. 产品与后续方向（纪要）

  • 六种画风模式（UI）与打印精细/标准/极速三档尚未在固件内做完整映射表；当前仍以一套全局 speed=5 为主，纸上模式列表用于你回传样张时对齐「当时选中的模式」。
  • 后续可按样张上的 S/E/C + 模式 + 画质主观评价，收敛为：模式默认档位表 + 图像特征（黑占比、细线）二次修正。

6. 编译与烧录

  • 构建命令：scons --beken=bk7252n -j4
  • 典型输出产物：
    • out/all_4M.1220.bin（整包）
    • out/rtthread_uart_4M.1220.bin（分区导出，视打包脚本而定）
