2026-05-19 WS热敏图预览压缩质量优化开发记录

需求背景

云端下发图片数据（MQTT binary, 1bpp 热敏位图, 384宽）经过压缩后在 245×245 预览框中显示，原有方案直接将 1bpp 源图缩放→抖动→1bpp 输出，压缩后的图片在屏幕上锯齿严重、细节丢失、抖动纹理明显，显示效果不理想。

优化目标

  • 提升缩略图预览的视觉质量：减少锯齿、保留细线条/文字、过渡平滑
  • 不增加过多内存开销（MCU 堆内存有限，原缓冲区 ~18KB）
  • 不改变对外 API（thermal_happy_set_ws_image_fit 接口不变）

技术方案演进

第1版：1bpp→1bpp（原有方案，效果差）

源 1bpp (384×N) → 均匀区域采样 → 灰度 0~255
→ 固定阈值 104 二值化 → Floyd-Steinberg 误差扩散
→ 输出 1bpp LV_IMG_CF_INDEXED_1BIT

问题：

  • 256 级灰度直接量化为 2 级（黑/白），误差扩散无法弥补
  • 均匀区域采样无边缘保护，文字和线条边缘模糊
  • 抖动纹理在 245×245 分辨率下肉眼可见

第2版：1bpp→4bpp（过渡方案，内存溢出）

源 1bpp → 中心加权区域采样 → 灰度 0~255
→ 1.2× 对比度增益 + 稀疏墨点增强
→ Floyd-Steinberg → 量化到 16 级灰度（4bpp）
→ 输出 LV_IMG_CF_INDEXED_4BIT（16色灰度调色板）
→ LVGL 直接渲染

改进：

  • 16 级灰度（白→浅灰8级→深灰8级→黑），量化误差降至原来的 1/8
  • 中心加权采样（块中心 ±1/4 半径权重 ×2）
  • 蛇形扫描（serpentine）改善误差分布

问题：

  • 缓冲区需要 ~31KB（LV_IMG_BUF_SIZE_INDEXED_4BIT(248,248) = 31064 bytes）
  • BK7252N 可用堆内存不足，rt_malloc 返回 NULL
  • 日志显示：[ui] ws image parse failed, len=18576

第3版：1bpp→2bpp（最终方案，质量与内存平衡）

源 1bpp → 中心加权区域采样 → 灰度 0~255
→ 1.2× 对比度增益 + 稀疏墨点增强
→ Floyd-Steinberg → 量化到 4 级灰度（2bpp）
→ 输出 LV_IMG_CF_INDEXED_2BIT（4色灰度调色板）
→ LVGL 直接渲染

优势：

  • 4 级灰度（白 → 浅灰 → 深灰 → 黑），比纯黑白提升明显
  • 缓冲区仅需 ~15.6KB（LV_IMG_BUF_SIZE_INDEXED_2BIT(248,248) = 15640 bytes）
  • 比原始 1bpp 方案的 ~18KB 还小，不会出现内存分配失败
  • Floyd-Steinberg 误差扩散保留，过渡自然

核心算法细节

1. 中心加权区域采样

当目标像素对应的源区域超过 1×1 像素时，采用中心加权计数：

源像素块中心 (cx, cy)，半径 (bw, bh)
- 距离中心 ≤ 1/4 半径：权重 ×2（边缘保护）
- 距离中心 > 1/4 半径：权重 ×1
- 最终：gray = (black_weighted * 255) / (total_weighted)

比均匀 box filter 更好地保留文字/线条边缘。

2. 稀疏墨点增强

对于含墨量极低的源块（如细笔画中文字符），额外提升灰度避免被量化吞没：

if (black * 16 < total)  gray += 3 * GRAY_STEP;  // < 6.25% 墨点
if (black * 8  < total)  gray += 2 * GRAY_STEP;  // < 12.5% 墨点

3. 2bpp 量化

// GRAY_STEP = 85 (255/3)
lvl = (v * 3 + 128) / 255;  // 0..3
error = v - lvl * 85;       // 误差用于扩散
Level理想灰度调色板 RGB
00 (白)(255,255,255)
185 (浅灰)(170,170,170)
2170 (深灰)(85,85,85)
3255 (黑)(0,0,0)

4. 自动反色检测

若输出图像 > 99.9% 像素为纯白（level 0），判定源数据反色，执行字节级 NOT 翻转。 2bpp 格式下 ~byte 天然正确翻转每个 2-bit 像素：00?11, 01?10。

5. 蛇形扫描（Serpentine）

偶数行从左到右扫描，奇数行从右到左，Floyd-Steinberg 误差扩散系数镜像对称，避免方向性伪影积累。

调色板反色问题修复

初版 4bpp 实现中调色板方向与渲染逻辑不匹配：

  • 渲染逻辑：gray=0（无墨）→ 量化 level 0 → 应显示白色
  • 初版调色板：pal[0] = RGB(0,0,0) 黑色 ?
  • 修正后：pal[0] = RGB(255,255,255) 白色 ?

#define THERMAL_PREVIEW_MAX_W   248U   // 缓冲区最大宽度（≥ 245 预览框）
#define THERMAL_PREVIEW_MAX_H   248U
#define THERMAL_GRAY_GAIN_Q8    307U   // 对比度增益 1.2× (Q8 定点)
#define THERMAL_SPARSE_BOOST_LO 3U     // 稀疏墨点增强 (< 6.25%)
#define THERMAL_SPARSE_BOOST_HI 2U     // 稀疏墨点增强 (< 12.5%)
#define GRAY_LEVELS             4U     // 2bpp: 4 级灰度
#define GRAY_STEP               85U    // 255 / (4-1)

新增内存分配失败诊断日志：

[thermal] alloc preview buf failed, size=31064

便于后续区分"解析失败"是内存不足还是数据格式问题。

待优化方向

  • 若后续硬件升级或堆管理优化后有更多可用内存，可重新启用 4bpp 获得更细腻的灰度表现
  • 可考虑根据源图尺寸动态分配缓冲区（节省小图时的内存）
  • 稀疏墨点增强的参数可进一步调优，针对中文字符笔画特征优化
