下面按固件里真实用法（thermal_printer.c 里 prn_tune set → s_prn.cfg / 位图循环）逐项说明你这组「激进第三组」在干什么。

prn_tune set line 1

  • 对应变量：line_cooling_us（行间冷却时间，单位 微秒）。
  • 含义：打完一行（走完一整行的移位、加热、步进节奏）之后，在进下一行前额外等的「晾一下」时间。
  • 1 的效果：几乎最短，纸走得快，但更容易出现：拖尾、粘连、边界发晕，细笔画有时也会显得「不干净」。
  • 注意：这里是 1 μs，不是关闭；若设为 0，代码里会用默认 DEF_LINE_GAP_US（比 1 大得多），反而不会像字面那样「零间隙」。

prn_tune set igap 0

  • 对应变量：inter_group_us（打印头按 4 组分时加热时，组与组之间的空隙，单位 μs）。
  • 含义：某一组加热点完歇一口气再处理下一组，给头和纸一点缓冲。
  • 0 在固件里的实际行为：sync_timing_cache_from_cfg() 里 inter_group_us == 0 会改用默认 DEF_GROUP_GAP_US（工程里是 30μs 量级），并不是「组间完全零间隔」。
  • 所以：你写 0 等价于「别用手写的 1μs，退回一组安全的默认值」，往往比 igap 1 略松一点。

prn_tune set motor 150

  • 对应变量：motor_step_us（电机每一步固定延时，单位 μs）。
  • 含义：纸往前走一小步要等多久；数越小走纸越快。
  • 150：固定高速步进；若配合 heat 偏低，容易出现 印迹偏淡（纸带走快、加热有效时间短）。
  • 若为 0：走内置加速表，不是你这组参数的思路。

prn_tune set heat 680 540 520

  • 对应变量：heat_us_at_5c、heat_us_at_25c、heat_us_at_40c（三个温度区间的基准加热时间，单位 μs）。
  • 含义：测温后插值选一根「基础加热」时间，再叠加按行黑点、dense、以及 print_line 里按组黑点做的微调。
  • 你这组数值：比常见「偏浓」配置（例如 880/840/720）明显更低 → 整体更淡、更省温，细线、浅灰、二维码小块更容易 发虚；若机器电压、纸张、头状态一般，这是最先会影响「清不清晰」的一项。

prn_tune set check 356

  • 对应变量：status_check_period_lines（每隔多少行做一次「测温 + 刷新加热基准缓存」）。
  • 含义：避免每一行都测 ADC，减 CPU 开销；数值越大，测温越稀疏。
  • 356：很长，大部分时间沿用上一次缓存的加热基准 → 吞吐好，但若打印很长、头温变化大，途中可能出现 浓度慢慢飘。
  • 若为 0：固件里 prn_tune set check 会改成 8（代码有特殊处理）。

prn_tune set white 296

  • 对应变量：white_run_max（连续空白行一次性「快进」最多合并多少行）。
  • 含义：位图里连续没有黑点的行，可以不按完整打印管线一行一行加热，而是连续步进多行赶纸。
  • 296：很大（上限文档写的是 512），大留白区域赶纸特别狠 → 总时间容易明显下降；若边界判断或机械上有抖动，极端情况下可能影响观感，但通常主要收益是速度。

prn_tune set yield 0

  • 对应变量：yield_period_lines（每隔多少行故意 rt_thread_mdelay(1) 一次）。
  • 含义：给别的线程一点 CPU，避免打印独占过久。
  • 0：关掉这项礼让 → 打印线程更「霸占」CPU，MQTT/UI 等更容易卡顿或饿死风险略升，但你这组追求的是极限吞吐。

prn_tune set dense 90 55 28 10 5

  • 对应变量：dense_boost_300 / _220 / _140 / _80 / _30（单位 μs，五个数按顺序对应）。
  • 含义：当一行里黑点个数超过阈值 >300、>220、>140、>80、>30 时，在已算好的基础加热上再多加这么多微秒，避免大块黑发灰。
  • 90 55 28 10 5：相对「浓」一点的默认（例如 135 90 55 28 10），你这组 整体补热更小 → 黑块、深色背景可能更浅；和 heat 三根偏低叠加时，清晰度/饱和度更容易不够。

prn_tune show

  • 含义：把当前 RAM 里这一套参数打印到串口，便于核对是否生效（打印中会拒绝修改）。

这组参数的「性格」一句话

极力追求速度（极短 line、快 motor、长 check、大白 batch、关 yield），整体加热偏保守（heat 低、dense 小）；清晰度若不够，优先怀疑 heat 三根和 motor，其次是 line 太短、dense 偏小。

补充：若是走 thermal_printer_print_text() 打的 ASCII 字，那段会临时切 内置 L1，不全吃你这组数；全吃这组的是 thermal_printer_print_bitmap 路径（含二维码图、云图等）。

下面按你文档 552–578 行里出现的顺序，逐项说明它们在固件里的含义（依据 applications/Printer/thermal_printer.c）。单位若无特殊说明均为 微秒 ?s。

prn_tune preset L1

调用 preset_speed_level_full(1)：载入 速度档位 1 对应的一套基础时序（apply_speed_profile）、按档位设的 位图打印默认（bitmap_tune_defaults_for_level）以及 高级默认（apply_advanced_defaults），再写回配置。
注意： 它会改掉一大片运行时参数；你后面紧跟的多条 set 是在这套基底上再覆盖，因此脚本里 preset 写在最前面是合理的。

prn_tune set clk 1

设置 clk_half_us：移位寄存器时钟 半周期延时。shift_line() 里每位数据会配合 clock 翻转，每次等待 clk_half_us（每位有效涉及多次该延时）。数值越小移位越快，过快可能丢数据；过大则整行移位变慢。

prn_tune set latch 2

设置 latch_us：latch 拉高后保持的时间，用于锁存一行点阵。过短可能锁不住；过长一般只是略拖慢。

prn_tune set motor 1150

设置 motor_step_us：步进电机 每一步之间的延时（当该值非 0 时，motor_delay() 固定用它，不再走加速表）。越大走纸越慢，通常有利于浓度和减少拖影；越小越快。

prn_tune set steps 2

设置 motor_steps_per_line（1–4）：在 seg（分段） 模式下，一行分成 4 组加热时，电机步进如何摊进去（motor_step_progressive）。whole 模式下整行打印走的是 feed_quick_two_steps()，这条对「每行几步」的表现影响方式不同，主要仍在 seg 路径体现。

prn_tune set line 240

同时写入 line_cooling_us（配置）和 line_gap_us（运行时）：表示 打完一行后、下一行前的行间间隔（冷却/节奏）。print_line_whole / print_line_segmented 末尾 delay_us(line_gap_us)（可能还会被 linegap 逻辑缩放或加点）。越大行距越大、热头更凉快；越小打得越快。

prn_tune set igap 36

设置 inter_group_us 并同步 group_gap_us：分段模式下，相邻两个「加热组」之间的间隔（有加热的组之后会 delay_us(group_gap_us)）。whole 模式下基本不走到这段。越大组间越疏，有利于散热和浓度稳定。

prn_tune set heat 1440 1280 1120

三个数依次为 heat_us_at_5c、heat_us_at_25c、heat_us_at_40c，并同步到运行时的 heat_5c_us 等。select_heat_us() 根据 ADC 估算的头温做 分段线性插值，得到当前行的基准加热时间。数值越大越浓，低温段通常用更大的基准。

prn_tune set heatlim 760 1680

设置 heat_lim_min_us 与 heat_lim_max_us（若顺序颠倒会自动交换）。clamp_heat_us() 把所有加热时间 钳在该区间内，避免过淡或过烫。你设的 760–1680 是最终允许的单脉宽（或 whole 下拆脉冲后的 每脉宽）的有效范围。

prn_tune set dense 280 200 120 65 25

按行黑点总数追加加热（在 thermal_printer_print_bitmap 里）：
黑点 >300 / >220 / >140 / >80 / >30 时，分别在基准 heat_us 上加 dense_extra_300 … dense_extra_30。用于 密图、大块黑 时加重，减轻发灰；过大易过热或糊。

prn_tune set check 8

status_check_period_lines：每隔多少行做一次 温度检测与过热处理，并刷新缓存的 heat_us_cached。设为 0 时代码里会 当成 8。你设为 8 即每 8 行检查一次。

prn_tune set white 0

white_run_max：0 表示关闭「连续空行批量快速走纸」。非 0 时，连续无黑点行最多合并跳过 white_run_max 行。二维码、大块留白图对纵向节奏敏感时，用 0 可避免跳过逻辑带来的行间不同步感。

prn_tune set yield 0

yield_every_n：0 表示关闭。非 0 时，每打印 n 行（白行跳过路径也会按行计数）rt_thread_mdelay(1)，把 CPU 让给其它线程。关掉打印更连贯、实时性更好；打开有利于系统其它任务。

prn_tune set stop 75 / prn_tune set resume 60

stop_temp_c / resume_temp_c：打印循环里周期性读温度，若 高于 stop，则阻塞等待直到 不高于 resume 再继续。用于 过热保护；间隔由上面的 check 决定。

prn_tune set log 1

log_level（0–3）：控制热敏驱动内部调试日志详细程度，级别越高打印越多（具体分级见 PRN_* 宏实现）。

prn_tune set strobe_guard 0 0

strobe_pre_us / strobe_post_us：每次拉高 STROBE（加热）前、后 的固定延时。适当加大可稳定驱动/采样时刻；0 表示不加额外防护间隔。

prn_tune set linegap_thr 48 120 220 300

四个阈值：lg_thr_sparse、lg_thr_mid、lg_thr_dense、lg_thr_vdense。
对 当前行黑点总数 line_black_dots：

  • ≤ 第一个 → 视为 sparse
  • ≤ 第二个 → mid
  • ≥ 第四个 → vdense
  • ≥ 第三个（且未到 vdense）→ dense
    用于下面 linegap_sparse_shift / mid_shift / dense_add / vdense_add 的分支。

prn_tune set linegap_sparse_shift 2

lg_sparse_shift（0–7）：当行属于 sparse（或整行无点时的冷却分支）时，对 line_gap_us 做右移缩小（line_gap_shift_us），即 行间变短、打得更快。数值越大缩得越狠（在代码含义上是 shift 位数）。

prn_tune set linegap_mid_shift 1

lg_mid_shift：mid 密度行的 line_gap_us 右移位数，介于 sparse 与 dense 之间。

prn_tune set linegap_dense_add 10

lg_dense_add_us：当行黑点达到 dense 档时，在 line_gap_us（及前述缩放结果）上 再加 的微秒数，给密行更多冷却时间。

prn_tune set linegap_vdense_add 25

lg_vdense_add_us：特密行（≥ lg_thr_vdense）再加的时间，通常 ≥ dense_add。

prn_tune set igap_sparse_shift 1

仅在 seg 模式有意义：某一组 没有加热 时，组间间隔用 group_gap_us >> igap_sparse_shift，比有加热的组更短，加快「空白组」节奏。whole 模式不走这段逻辑。

prn_tune set heatmode whole

heat_layout = WHOLE_LINE：一整行 384 点 一次移位进移位寄存器，再按 heatpulse 做整行 STROBE（print_line_whole）。与 seg（四段分别移位+加热）相对。

prn_tune set heatpulse 6

whole_line_strobe_pulses（1–15）：whole 模式下把一行理论加热量拆成 多次短脉冲，脉冲之间固定 14?s；单次脉宽约为「总目标加热时间按脉冲数均分」再经 heatlim 钳位。增大脉冲数有利于减轻瞬时电流、改善大块实心，但总时间上会有额外间隙。

prn_tune set gshape 8 130 16 80 28 35 40 82 110 70 65 56 35

13 个整数写入 prn_gshape_t 并 gshape_enable = true。在 seg 模式下，按 每组黑点个数 微调该组 group_heat_us（黑点少则减时、多则加时），见 print_line_segmented 里对 gs->t8/s8 … t56/a56 的分支。

重要： 当前脚本是 heatmode whole，整行打印走 print_line_whole，不会执行上述 gshape 分支。这条命令对 whole 实际打印 基本无影响；若要用 gshape，需 heatmode seg。

prn_tune show

打印当前所有关键运行时参数摘要（speed、clk/latch、motor、heat、white、linegap、heatmode、pulses、gshape 等），用于核对脚本是否生效。

如需把「whole 下也想按区域调加热」这类需求对齐到代码路径，需要另做逻辑扩展；当前固件里 gshape 仅绑定 seg。
